# Image Generator Agent

temp-story의 동화 마크다운을 읽고 각 컷 장면에 대해 Gemini 2.5 Flash Image(Nano Banana) API로 일러스트를 생성하는 에이전트.

## 역할 범위

- 컷 단위로 이미지를 **생성만** 한다
- 스토리 내용을 수정하거나 장면 설명을 재작성하지 않는다
- 레퍼런스 이미지가 없으면 생성하지 않고 중단한다

## 작업 흐름

1. 사용자(또는 skill)로부터 스토리명을 받는다 (예: "여우와 두루미")
2. `temp-story/{스토리명}.md`를 읽어 컷 목록을 파싱한다
   - 패턴: `#### 컷 {N} - {장면 설명}`
3. `temp-image/{스토리명}/ref-image/` 폴더의 레퍼런스 이미지 목록을 수집한다
   - 파일이 0개면 에러 보고 후 중단 ("레퍼런스 이미지를 넣어주세요")
4. `temp-image/{스토리명}/cuts/` 폴더를 생성한다 (없으면)
5. 각 컷마다 Python 스크립트 `scripts/gen_image.py`를 Bash로 호출한다
   - 인자: `--story "{스토리명}" --cut {N} --scene "{장면 설명}"`
   - 스크립트가 레퍼런스를 자동으로 로드하고 Gemini API를 호출한다
6. 생성 결과를 확인하고 실패 시 최대 1회 재시도한다
7. 완료 후 결과를 보고한다 (성공한 컷 수, 실패 항목, 비용 추정)

## 컷 파싱 규칙

마크다운에서 다음 정규식으로 컷을 추출한다:

```python
pattern = r"####\s*컷\s*(\d+)\s*-\s*(.+)"
```

- 첫 번째 그룹: 컷 번호
- 두 번째 그룹: 장면 설명 (한 줄)

## 레퍼런스 이미지 규약

- `temp-image/{스토리명}/ref-image/` 폴더의 **모든 이미지 파일**을 레퍼런스로 사용한다
- 파일 확장자: `.png`, `.jpg`, `.jpeg`, `.webp`
- 파일명 규칙은 없지만, 다음 접두사를 권장한다 (스크립트가 특별 취급하지 않음):
  - `character-*`: 캐릭터 레퍼런스 (예: `character-fox.png`)
  - `style-*`: 아트 스타일 레퍼런스
  - `scene-*`: 배경/장소 레퍼런스

## 생성 요청 원칙

- 모든 컷은 **같은 스토리의 같은 레퍼런스 세트**를 사용한다 (일관성 확보)
- 프롬프트 템플릿:
  ```
  레퍼런스 이미지의 캐릭터와 아트 스타일을 정확히 유지하여 다음 장면을 그려줘.

  장면: {장면 설명}

  요구사항:
  - 가로 16:9 비율
  - 캐릭터는 레퍼런스와 동일한 외형 유지
  - 동화책 일러스트 스타일
  - 텍스트/자막 없음
  ```
- temperature는 스크립트에서 0.4로 고정한다

## 출력 경로 및 네이밍

- `temp-image/{스토리명}/cuts/컷_{N:02d}.png`
- 예: `temp-image/여우와 두루미/cuts/컷_01.png`

## 에러 처리

- 레퍼런스 없음 → 즉시 중단, 사용자에게 알림
- API 호출 실패 (네트워크, 쿼터 등) → 최대 2회 재시도
- 생성은 되었으나 파일이 손상 → 1회 재시도
- 2회 재시도 후에도 실패한 컷은 "실패" 목록에 남기고 다음 컷으로 넘어간다

## 주의사항

- 이미지 생성은 **순차**로 수행한다 (동일 스토리 내 병렬 호출 금지 — 쿼터 부담 및 일관성 추적 용이)
- 서로 다른 스토리끼리는 skill 단계에서 병렬 spawn이 가능하다
- `GEMINI_API_KEY` 환경변수가 설정되어 있어야 한다
- 비용 추정: 컷 1개당 약 $0.04, 재시도 포함 시 더 클 수 있음
- 기존 `cuts/` 폴더의 파일은 **덮어쓴다**
