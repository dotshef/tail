# PDF Generator Agent

tail-bookform의 최종본을 HTML로 변환한 뒤 Playwright를 사용하여 PDF로 생성하는 에이전트.

## 역할 범위

- bookform 마크다운을 읽어 스타일이 적용된 HTML을 생성한다
- Playwright로 HTML을 PDF로 변환한다
- `<!-- expressions -->` 블록은 무시한다 (PDF에 포함하지 않음)

## 입력

스킬로부터 다음을 전달받는다:
- 스토리명 (예: `여우와 두루미`)
- 대상 언어 코드 (예: `en`, `zh-tw`, `jp`)

입력 파일: `tail-bookform/{lang}/{스토리명}.md`

## PDF 레이아웃

### 1. 표지 (첫 페이지)

- 제목(`#`)을 페이지 중앙에 크게 배치한다
- `<sub>` 번역을 제목 아래에 작은 글씨로 배치한다
- 표지에는 본문을 넣지 않는다
- 표지 후 페이지 브레이크

### 2. 본문 (둘째 페이지부터)

- `##` 장 제목이 나오면 새 페이지에서 시작한다
- `<sub>` 번역을 장 제목 아래에 작은 글씨로 배치한다
- `### N 페이지 (N단어)` 헤딩은 PDF에 표시하지 않는다 (숨김 처리)
- 이미지(`![컷 N](...)`)는 해당 위치에 배치한다
  - 이미지 경로는 절대 경로로 변환한다 (`c:/D/tail/` 기준)
  - 이미지 너비는 페이지 폭의 80%로 설정한다
  - 이미지 아래에 적절한 여백을 둔다
- 이미지 아래에 본문 텍스트가 이어진다
- `<!-- expressions -->` 블록은 무시한다

### 3. 폰트 및 스타일

- 본문: 명조체 (serif) — `'Noto Serif KR', 'Batang', serif`
- 제목: 명조체, 볼드
- `<sub>` 번역: 본문보다 작은 크기, 회색 계열
- 본문 글자 크기: 14px
- 줄간격: 1.8
- 대사(큰따옴표로 시작하는 줄): 들여쓰기 없이, 본문과 동일 스타일

### 4. 페이지 설정

- A5 크기 (148mm × 210mm) — 동화책에 적합
- 여백: 상하 20mm, 좌우 15mm

## 생성 과정

1. bookform 마크다운을 읽는다
2. 마크다운을 파싱하여 HTML로 변환한다
   - frontmatter는 무시한다
   - `<!-- expressions -->` 블록은 제거한다
   - `### N 페이지` 헤딩은 제거한다
   - 이미지 경로를 절대 경로(`file:///`)로 변환한다
3. CSS 스타일을 적용한 완전한 HTML 파일을 생성한다
4. HTML 파일을 `tail-pdf/{lang}/{스토리명}.html`에 저장한다
5. 프로젝트 루트의 `generate-pdf.py`를 실행하여 HTML → PDF 변환한다
   ```bash
   python c:/D/tail/generate-pdf.py {lang} {스토리명}
   ```
   - 스크립트가 새로 작성하지 않는다 — 기존 `generate-pdf.py`를 호출만 한다
6. PDF 생성 결과를 확인하고 보고한다

## 출력

- `tail-pdf/{lang}/{스토리명}.pdf`
- `tail-pdf/{lang}/{스토리명}.html` (중간 산출물)

## 주의사항

- 이미지 경로는 반드시 절대 경로(`file:///c:/D/tail/...`)로 변환해야 PDF에서 표시된다
- 한국어 텍스트가 깨지지 않도록 UTF-8 인코딩을 명시한다
- Playwright의 `page.pdf()`는 `@page` CSS를 무시하므로, 페이지 크기/여백은 `pdf()` 옵션으로 설정한다
- CSS에서는 `page-break-before`, `page-break-after`로 페이지 브레이크를 제어한다
- PDF 생성 후 파일 존재 여부를 확인하고 결과를 보고한다
